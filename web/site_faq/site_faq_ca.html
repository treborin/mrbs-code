Com puc iniciar la sessió?

El sistema es pot configurar per utilitzar un dels diversos mètodes d'autenticació, incloent LDAP, Netware i SMB. Consulteu el vostre administrador del sistema si ho feu Teniu problemes per iniciar sessió. Algunes funcions estan restringides a determinades usuaris, i altres usuaris rebran el missatge No teniu drets d'accés modificar aquest element. Consulteu el vostre administrador del sistema si això no funciona correctament per a tu. Si el sistema està configurat per utilitzar l'autenticació LDAP, això vol dir que vostè inicieu sessió amb el mateix nom d'usuari i contrasenya que utilitzeu per rebre el correu electrònic p. ex. Mark Belanger i MyPassword.

Per què no puc suprimir/alterar una reunió?

Per suprimir o modificar una reunió, heu d'iniciar sessió com a mateixa persona que va fer la reunió. Contacta amb un de la sala de reunions administradors o la persona que inicialment va fer la reunió fer-lo esborrar o canviar.

Amunt

Com puc crear una reunió?

Si feu clic a l'hora desitjada, us portarà a la pantalla de reserva. Ompliu els detalls i feu clic a "Desa".

En lloc de fer clic a l'hora desitjada, podeu seleccionar un interval de temps i també una habitació o un interval de dies arrossegant el cursor per sobre d'un grup de cel·les del visualitzacions diàries o setmanals. Quan deixeu anar el ratolí, se us portarà a pantalla de reserva i les franges horàries i l'habitació o interval de dies hauran estat omplert per tu.

Quina diferència hi ha entre cinquè i últim?

Quan feu una reserva periòdica mensual, el cinquè dia feiner de cada mes és només això. Si un mes no té un cinquè dia laborable, no hi haurà reserva fet per a aquell mes. Si trieu el últim dia feiner del mes, a la reserva es farà el quart dia laborable si no hi ha un cinquè dia laborable.

Finth last funciona de manera similar, però comptant enrere des del final de el mes. Si feu servir primer, us donarà el quart darrer dia feiner del mes si no hi ha un cinquè l'últim dia feiner.

Com puc suprimir una instància d'una reunió recurrent?

Seleccioneu el dia/habitació/hora que voleu suprimir i seleccioneu Suprimeix l'entrada.

Com puc programar sales a diferents llocs?

Tu no. Actualment el sistema no pot reservar dues habitacions diferents simultàniament. Cal programar cadascun per separat. Assegureu-vos que el temps que voleu està disponible als dos llocs abans de fer una reserva.

La meva reunió no s'ha pogut crear perquè hi ha massa entrades!

Cap reunió pot crear més de 365 entrades. Cal que hi hagi un límit en el nombre de reunions creades. Aquest nombre es pot augmentar si cal.

Què passa si diverses persones programen la mateixa reunió?

La resposta breu és: la primera persona a fer clic el botó Envia guanya. Darrere de les escenes, el sistema utilitza un multiusuari adequat i multifils base de dades relacional que pot gestionar molts milers d'usuaris simultanis.

Amunt

Quina diferència hi ha entre Intern i Extern?

Per defecte, MRBS defineix dos tipus de reunió. Intern significa que el a la reunió només hi assistiran els treballadors. Una reunió externa També hi poden assistir clients, venedors, inversors, etc. El vostre lloc pot definir fins a un total de 10 tipus de reunions, segons les vostres necessitats. Les reunions es destaquen a la vista del calendari principal amb un color corresponent al seu tipus i a la part inferior es mostra una clau de color de tots els tipus definits de la vista principal del calendari.

Com funciona el sistema i qui l'ha escrit?

El Sistema de reserves de sales de reunions és un programari de codi obert que es distribueix sota la Gnu Public License (GPL). Això vol dir que el programari és lliure d'utilitzar, distribuir i modificar.

El sistema està escrit principalment en PHP, que és un llenguatge de programació de codi obert que es pot incrustar a la web pàgines similars en concepte a les pàgines del servidor actiu de Microsoft. PHP és especialment bo per accedir a bases de dades.

La base de dades utilitzada per al sistema és qualsevol MySQL o PostgreSQL. MySQL és un sistema molt ràpid, multifils, multiusuari i robust Servidor de bases de dades SQL (Structured Query Language) que també és GPL. PostgreSQL és un Object Relational SQL de codi obert multiusuari amb totes les funcions servidor de bases de dades.

El sistema funcionarà en diverses plataformes, inclosa l'arquitectura de PC utilitzant el sistema operatiu Linux. Linux, és un sistema operatiu gratuït, de codi obert, semblant a Unix.

El servidor web que s'utilitza és una altra peça de codi obert i gratuït programari. El servidor web Apache és el servidor web més popular del món.

La conclusió és: cada peça d'aquest sistema, des del sistema operatiu fins a l'aplicació, és completament gratuït: codi font i tot.
